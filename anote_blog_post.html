Did the AI Do It Right, or Just Get Lucky?

Why evaluating autonomous AI agents on outcomes alone is no longer enough — and what we should measure instead.

AI Evaluation Agentic Systems 7 min read
The Problem

The right answer, the wrong way

Imagine hiring a financial analyst who hands you the correct report every Monday morning, but you later discover they've been guessing randomly, occasionally deleting files, and querying the wrong database throughout the week. The output looked fine. The process was a disaster.

This is the exact problem facing AI agents today. As large language models move from answering questions to actually doing things — querying databases, investigating fraud cases, retrieving documents, triggering workflows — we've kept using the same old report card: did it get the right answer at the end?

Most benchmarks measure whether the agent eventually reaches the correct final answer. But in real-world deployments, the path matters as much as the destination.

Why It Matters

Agents fail in ways that don't show up in accuracy scores

Real-world agent failures are subtle. An agent might select the wrong retrieval tool but stumble onto the right document anyway. It might hallucinate a tool call that doesn't exist, recover gracefully, and still deliver a correct result. It might forget a constraint from five steps ago — say, "only flag transactions over $10,000" — and produce an answer that looks right but violates the original task specification.

These failures are invisible to outcome-only benchmarks. But in production environments — especially in finance, healthcare, legal, or compliance workflows — they represent unacceptable risk. A system that occasionally reaches correct answers through unreliable processes is not a system you can trust.

The Research

Introducing process-level evaluation for LLM agents

This paper proposes a new benchmark — tentatively called AgentTrace — designed to evaluate autonomous agents at every step of their decision-making trajectory, not just at the finish line.

The benchmark simulates realistic enterprise workflows across domains like fraud investigation, financial compliance, incident triage, and internal document retrieval. Each task requires the agent to move through multiple stages: understanding user intent, selecting tools, executing them, interpreting results, and recovering from any mistakes that occur along the way.

Crucially, the benchmark captures the full agent trajectory — every tool call, every intermediate state, every self-correction — and evaluates the quality of each decision independently.

The Metrics

Six things we should actually be measuring

Tool selection accuracy
Did the agent choose the right tool for the right sub-task?
Hallucinated action rate
How often did the agent invoke tools or resources that don't exist?
Recovery score
When errors occurred, did the agent recognize and correct them?
State consistency
Did the agent maintain constraints and context across all steps?
Human intervention cost
How much manual correction was needed before safe deployment?
Unsafe action rate
Did the agent attempt high-stakes actions without human approval?
The Bigger Picture

Trust is a process problem, not an accuracy problem

The broader ambition here is to shift the standard for what "a good AI agent" means. As organizations begin deploying autonomous systems in high-stakes workflows, accuracy on a held-out test set is not sufficient evidence of trustworthiness. What matters is whether the agent behaves reliably — across steps, across failures, across edge cases — and whether humans can intervene meaningfully when it doesn't.

By measuring process-level reliability and human-in-the-loop correction costs, this work aims to provide a more honest benchmark for the question every enterprise deploying AI agents should be asking: not "does it work?" but "can we trust it?"

The question was never whether AI agents can get the right answer. The question is whether they get there in a way that is safe, consistent, and correctable — every single time.
